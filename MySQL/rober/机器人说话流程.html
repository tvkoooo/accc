机器人说话流程

需要用到配置常量

  1. ROBOT_TALK_INTERVAL_TIME 230    1   0   分钟
  2. ROBOT_TALK_GIFT_MIN_GOLD 231    50  0   金币

mysql的数据结构

  1. /*!40101 DROP TABLE IF EXISTS `t_robot_talking_base`*/;
  2. /*!40101 SET @saved_cs_client     = @@character_set_client */;
  3. /*!40101 SET character_set_client = utf8 */;
  4. CREATE TABLE `t_robot_talking_base` (
  5.     `talk_id` int(11) unsigned NOT NULL AUTO_INCREMENT comment '语句id',
  6.     `talk_topic` int(8) NOT NULL  comment '用户话题，采用数字计数；1代表普通类；2代表新人进场；3礼物语',
  7.     `talk_string` char(100) NOT NULL  comment '说话内容',
  8.     PRIMARY KEY (`talk_id`)
  9. ) ENGINE=InnoDB AUTO_INCREMENT=52010000 DEFAULT CHARSET=utf8 comment='用户配置数据表';
  10. /*!40101 SET character_set_client = @saved_cs_client */;

Redis的数据结构

  1. redis 缓存talk_topic话语结构：  redis_hash
  2. (redis)hash robot:talk:mysql:hash:%u(uint32 talk_topic)
  3. {
  4.     f(uint32 talk_id),
  5.     v(string talk_string),
  6. }
  7. (cache)EXPIRE 3600;
  1. redis 缓存talk_topic索引结构： redis_set
  2. (redis)set robot:talk:mysql:set:%u(uint32 talk_topic)
  3. {
  4.     k(uint32 talk_id),
  5. }
  6. (cache)EXPIRE 3600;
  1. redis 所有房间，某房间内任意用户最后一次说话时间点结构： redis_time
  2. (redis)hash robot:talk:room:lasttalk:hash
  3. {
  4.     f(uint32 sid(房间号)),
  5.     v(uint32 timecode_now),
  6. }
  7. (cache)EXPIRE -1;
  1. redis 机器人列表结构： redis_robot
  2. (redis)zset robot:talk:room:member:zset:%u（uint32 房间号  %1024取模）
  3. {
  4.     k(uint32 uid(用户号)),
  5.     score(uint32 timecode_now),
  6. }
  7. (cache)EXPIRE -1;

逻辑流程

1.刷新redis： mysql 数据 读取 加载到 redis 缓存 redis_hash 和 redis_set

  1. 去数据库select 数据表，放入redis。
  2. 刷新数据库的人，需要删除 redis_hash 或者 redis_set 缓存，让系统重新刷新（或者自己维护redis缓存）
  3. del robot:talk:mysql:hash:1  删除普通话redis
  4. del robot:talk:mysql:hash:2  删除新人redis
  5. del robot:talk:mysql:hash:3  删除礼物语redis

2.清空redis： 清空redis对应 talk_topic 的索引和话语

  1. 去数据库select 数据表，放入redis。
  2. 刷新数据库的人，需要删除 redis_hash  redis_set 缓存（或者自己维护redis缓存）

3.取数据流程： redis_set 先随机出 talk_id，由talk_id 取出redis_hash 中的话语 返回话语

  1. 先 redis_set 用
  2.     SRANDMEMBER robot:talk:mysql:set:%talk_topic
  3. 随机取出k值,
  4. 再根据k去 redis_hash 取
  5.     hget robot:talk:mysql:hash:%talk_topic k
  6. 说话内容。
  7. 如果redis缓存数据消失
  8.     先进行功能2清空redis，再进行功能1，刷新redis。重新取数据

4.redis缓存存储机器人id过程（房间号）：根据房间号，redis_robot 缓存 机器人id

  1. 记录此时刻，添加入缓存redis_robot
  2. ZADD  robot:talk:room:menmber:zset:%u（uint32 房间号） timecode_now 用户信息.uid

5.挑选机器人过程（房间号）：根据房间号，去redis 缓存redis_robot 获取机器人id

  1. //备注机制：缓存redis_robot会以时间顺序记录进入该房间的所有机器人id，最开始进入的机器人如果未出房间，由最开始机器人说话，如果该机器人出房间，选用剩下排序第二个机器人来说话，类推。
  2. 挑选 ZRANGE robot:talk:room:menmber:zset:%u（uint32 房间号） 0 0
  3. 获取列表中的第一个。

8.用户入场流程（房间号，机器人用户信息，主播用户信息，目标用户信息，说话具体内容）

  1. 检查是不是机器人
  2.     是：执行步骤4.redis缓存机器人过程（房间号） 
  3. 检查是不是新人
  4.     是：机器人挑选过程（房间号）,当做发言机器人.
  5.         user_topic=2; @用户   
  6.         取redis数据流程(talk_topic),当做说话具体语言
  7.         @人的具体实现过程（&返回结果，房间号，机器人用户信息，主播用户信息，目标用户信息，说话具体内容）

9.用户出场流程(用户信息,房间号)

  1. 如果是机器人，移出该房间对应机器人id 的缓存
  2. 移除 ZREM robot:talk:room:menmber:zset:%u（uint32 房间号） 用户信息.uid

10.礼物语流程（&返回结果，房间号，主播信息，进入用户信息，金币值）

  1. user_topic=3; 
  2. 如果 礼物信息.金币 大于 ROBOT_TALK_GIFT_MIN_GOLD=50金币
  3.     机器人挑选过程（房间号）,当做发言机器人.
  4.     取redis数据流程(talk_topic),当做说话具体语言
  5.     普通说话的具体实现过程（&返回结果，房间号，机器人用户信息，主播用户信息，说话具体内容）

11.普通类流程(&返回结果，房间号，主播信息)// 查看触发时间差是否满足设定时间，满足则说话

  1. user_topic=1; 
  2. 取出本房间号最后 真人 说话时间 
  3. 时间差（当前时间-取出时间）
  4. 判断时间差      是否      大于    设置时间（1分钟=60s）
  5. 如果是：
  6.     从房间机器人当中取出机器人作为发言
  7.     从redis取出说话内容
  8.     普通说话具体实现
  9.     重置说话时间

12.robot 倾听机制(房间号，用户发言信息)

  1. 判断是否是机器人，如果是，跳出该环节
  2.     如果不是机器人：
  3.     记录该房间号的真人说话时刻listen_time（用于后续比对）

13.主播入场流程(主播信息,房间号)

  1. 主播入场，设置该主播房间号sid无人说话时间戳

14.主播离场流程(主播信息,房间号)

  1. 移除对应主播房间号sid数据
  2. 清除对应房间号的机器人id的redis缓存
